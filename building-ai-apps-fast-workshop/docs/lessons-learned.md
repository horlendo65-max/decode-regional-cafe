# Lessons Learned

## 1. AI apps are more than model calls

A useful AI product needs clear user experience, safe prompts, API boundaries, deployment and iteration.

## 2. Agents help organise decision-making

Agents should coordinate the solution. They should not become giant files containing every feature.

## 3. Skills should be small and reusable

A good skill does one thing well. For example, parsing M-PESA text should be separate from writing budget advice.

## 4. Workflows turn capabilities into products

A workflow connects skills into a useful journey. This is where an idea becomes a product experience.

## 5. Prompts are product assets

Prompts should be versioned, reviewed and improved like code. Clear prompts produce better and safer results.

## 6. Hosted models are excellent for rapid MVPs

Free hosted APIs such as Gemini reduce setup time and let participants focus on product value.

## 7. Deployment changes the conversation

A deployed URL makes the MVP real. It enables feedback, demos and iteration.

## 8. Responsible AI still matters

For finance, health and agriculture, the app should avoid pretending to be a certified expert. It should provide guidance, explain uncertainty and recommend professional help where appropriate.

## 9. Reuse beats rewriting

The same architecture can support Expense, Health, Agriculture or Education apps by changing agents, skills and workflows.

## 10. AI coding tools are collaborators, not owners

Coding assistants can scaffold fast, but the developer remains responsible for architecture, security, correctness and deployment.
